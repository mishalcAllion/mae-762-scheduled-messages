Cross-Cutting Dependency — Timezone Handling

The Command Center team spans multiple timezones. This must be resolved before any scheduling story goes to dev.

Server

UTC

All timestamps stored and processed in UTC on the backend

Agents

Philippines — UTC+8

Sri Lanka / Ceylon — UTC+5:30

Central Time — UTC−6

Required Behaviour

  • — Picker shows agent's local time
  • — List view shows local time + UTC offset
  • — Client's local time shown alongside
  • — Server always receives UTC

Acceptance Criteria (Epic Level)

AC1 Agent can schedule a message from the chat composer using a date + time picker
AC2 Scheduled message appears in the thread with a visible "Scheduled" badge and timestamp
AC3 Agent can edit or cancel a scheduled message from within the thread
AC4 Dedicated list view shows all pending scheduled messages across all clients
AC5 Agent can edit or cancel any message from the list view without navigating to the thread
AC6 All times shown to agents in their local timezone — server stores UTC
AC7 Any agent can see and manage a scheduled message set by another agent on the same thread
AC8 If delivery fails, agent receives an in-app notification and can retry from the list view
